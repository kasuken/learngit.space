Advanced Log Filtering Techniques

Basic git log shows you a timeline, but advanced filtering transforms it into a precision investigation tool. Master these techniques to extract exactly the intelligence you need.

Time-Based Investigation

Temporal Filtering

# Investigate specific time periods
git log --since="2024-01-01" --until="2024-12-31"

git log --since="3 months ago" --until="1 week ago"

# Precise temporal investigation
git log --since="2024-10-15 14:30" --until="2024-10-15 18:00"
git log --after="Oct 15 2024" --before="Oct 16 2024"

# Relative time investigation

Author & Contributor Analysis

# Track specific operatives
git log --author="Sarah Chen"
git log --author="sarah@"  # Email partial match
git log --committer="jenkins@ci"  # CI system commits

# Multiple author investigation
git log --author="Sarah\|John\|Maria"  # Regex OR pattern
git log --grep="hotfix" --author="emergency@"

# Exclude specific contributors
git log --perl-regexp --author="^(?!.*jenkins).*"

Content-Based Forensics

# Combined intelligence gathering

Git Bisect: Bug Hunter Protocol

EXPERT LEVEL

Mission: Systematically hunt down the exact commit that introduced a bug

Binary Search Investigation

1
Initialize Investigation
# Start bisect investigation
git bisect start

# Mark current commit as bad (bug exists)
git bisect bad

# Mark known good commit (bug didn't exist)
git bisect good v2.1.0  # or specific commit hash
2
Test & Mark Evidence
# Git automatically checks out middle commit
# Test your application/run tests
npm test  # or your test command

# Mark result based on evidence
git bisect good    # If bug doesn't exist here
git bisect bad     # If bug exists here
git bisect skip    # If can't determine (build fails, etc.)
3
Automated Hunt Protocol
# Automate the entire investigation
git bisect run npm test  # Run until test passes
git bisect run ./test-script.sh  # Custom test script
git bisect run python -m pytest tests/security_test.py

# Advanced automated investigation
git bisect run sh -c "make && ./run-integration-tests"
4
Close Investigation
# Git shows you the guilty commit
git bisect reset  # Return to original HEAD

# Investigate the culprit
git show [guilty-commit-hash]
git log -p [guilty-commit-hash]

Statistical Crime Analysis

TACTICAL LEVEL

Mission: Extract patterns and statistics from repository history for strategic intelligence

Contributor Analysis

# Top contributors by commit count
git log --format='%aN' | sort | uniq -c | sort -nr | head -10

# Contributor activity by time period
git log --since="6 months ago" --format='%aN %ad' --date=short | \
    awk '{print $1, $2}' | sort | uniq -c | sort -nr

# Lines of code by contributor (approximation)
git log --numstat --format='%aN' | \
    awk 'NF==3 {plus+=$1; minus+=$2} NF==1 {name=$0} END {print name, plus, minus}'

Temporal Pattern Analysis

# Commits by day of week
git log --format='%ad' --date=format:'%w' | sort | uniq -c

# Commits by hour of day (find peak productivity)
git log --format='%ad' --date=format:'%H' | sort | uniq -c | sort -k2n

# Monthly activity trends
git log --format='%ad' --date=format:'%Y-%m' | sort | uniq -c

# Commit size patterns
git log --oneline --numstat | \
    awk '/^[a-f0-9]/ {files=0; added=0; deleted=0} \
         /^[0-9]/ {files++; added+=$1; deleted+=$2} \
         /^$/ {print files, added, deleted}'

File Evolution Intelligence

# Most frequently modified files (hotspots)
git log --name-only --format='' | sort | uniq -c | sort -nr | head -20

# File modification by contributor
git log --name-only --format='%aN' -- src/critical.js | \
    grep -v '^$' | sort | uniq -c | sort -nr

# Track file size evolution
git log --oneline --numstat -- important-file.js | \
    awk '/^[a-f0-9]/ {commit=$0} /^[0-9]/ {print commit, $1+$2}'

Case 2: The Elusive Performance Bug

CRITICAL PRIORITY

Evidence Summary

  • Issue: Application performance degraded dramatically in recent weeks
  • Timeline: Performance was good in version 1.5, terrible in current version
  • Impact: Response times increased from 100ms to 3000ms
  • Your Mission: Use git bisect to find the exact commit that caused the slowdown
1

Create Performance Crime Scene

# Create repository with performance regression
mkdir performance-investigation
cd performance-investigation
git init

# Create initial "fast" implementation
cat > performance-test.js << 'EOF'
// Fast implementation - O(1) lookup
const cache = new Map();

function fastLookup(id) {
    if (cache.has(id)) {
        return cache.get(id);
    }
    const result = `Result for ${id}`;
    cache.set(id, result);
    return result;
}

// Performance test
const start = Date.now();
for (let i = 0; i < 10000; i++) {
    fastLookup(i % 100);
}
const end = Date.now();
const time = end - start;
console.log(`Execution time: ${time}ms`);
process.exit(time > 1000 ? 1 : 0);  // Exit 1 if slow (>1000ms), 0 if fast
EOF

git add performance-test.js
git commit -m "v1.5: Initial fast implementation with caching"
git tag v1.5

# Add some innocent commits
echo "// Added documentation" >> performance-test.js
git add performance-test.js
git commit -m "Add documentation comments"

echo "// More documentation" >> performance-test.js
git add performance-test.js
git commit -m "Improve code documentation"
2

Introduce the Performance Bug

# Replace with slow implementation (the bug!)
cat > performance-test.js << 'EOF'
// Slow implementation - O(n) lookup every time
const data = [];

function slowLookup(id) {
    // Someone "improved" this by removing the cache... badly
    for (let i = 0; i < 1000; i++) {  // Simulate expensive operation
        data.push(`Item ${i}`);
    }
    
    // Linear search through growing array
    for (let item of data) {
        if (item.includes(id.toString())) {
            return `Result for ${id}`;
        }
    }
    return `Result for ${id}`;
}

// Performance test
const start = Date.now();
for (let i = 0; i < 10000; i++) {
    slowLookup(i % 100);
}
const end = Date.now();
const time = end - start;
console.log(`Execution time: ${time}ms`);
process.exit(time > 1000 ? 1 : 0);  // Exit 1 if slow (>1000ms), 0 if fast
EOF

git add performance-test.js
git commit -m "Refactor lookup function for better maintainability"

# Add more commits to hide the evidence
